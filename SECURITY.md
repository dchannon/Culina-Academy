# Security

## Reporting

If you believe the public documentation or examples expose sensitive information, contact the Data Culina support or security owner directly. Do not open a public issue with secrets, credentials, private endpoint details, or sensitive client data.

## Handling Evidence

When preparing diagnostic evidence:

- remove passwords, tokens, private keys, and secret values
- avoid attaching full personal data extracts
- include only the metadata, status, failure message, and run context needed for diagnosis
- sanitize screenshots when they show sensitive values

## Scope

This repository contains public documentation, examples, and optional workbook exercises. It does not publish the Culina runtime package.
