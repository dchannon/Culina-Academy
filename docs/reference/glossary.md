# Glossary

## Control Plane

The configured and operational record of framework-managed work.

## Orchestration Plane

The runtime coordination layer that plans, checks readiness, dispatches, monitors, and finalizes work.

## Data Plane

The layers where source data is received, standardized, transformed, persisted, and served.

## Workflow

A configured unit of framework-managed work.

## Run

One runtime instance of workflow execution.

## Dependency

A required upstream condition that must be satisfied before work can run.

## Dispatch

The action of sending ready work to an execution handler.

## Landing

The first framework-controlled intake layer for source input.

## Delta

The standardized intake layer after raw input is normalized.

## Staging

The standardized working layer used before transformation into modeled structures.

## Integration

The transformation alignment layer before durable persistence.

## EDW / Core Warehouse

The durable modeled warehouse layer.

## Consumption

The output-serving layer used by reports, downstream interfaces, or users.
